- (technique) started Job to Be Done
- (insight by user) The primary job is to find videos likely to contain useful information; extraction is secondary.
- (idea by coach) Treat discovery as a high-recall candidate funnel, then prioritize candidates for human review rather than claiming the system can verify their facts automatically.
- (insight by user) Both practical facts and experiential insights matter; practical facts can become stale and experience reports can conflict, so YouTube must reuse the existing Facebook capture safeguards.
- (insight by coach) Current knowledge pipeline already handles this: immutable readable capture versions, evidence-backed AI extraction and independent judgment, freshness/verification states, conflict relations, risk-prioritized operator review, and retrieval exclusion or caveat-only policy.
- (decision by user) MVP target is transcript-based capture; AI should score available metadata/transcript and may immediately analyze the video itself when the score is high, following an AI-first mechanism analogous to Facebook capture.
- (idea by coach) Use a staged AI-first funnel: API metadata and transcript triage, deterministic budget/risk gates, multimodal video analysis only for high-value candidates, then the existing evidence/judgment/conflict/review lifecycle; multimodal analysis creates evidence candidates, never an automatically trusted fact.
- (insight by user) A YouTube capture pipeline already uses AI to analyze and extract whole videos in 30-minute segments; selective analysis of high-signal segments is a promising optimization.
- (idea by coach) Keep full segmented analysis as the completeness path while adding targeted segments as an AI-selected fast path; discovery should decide which path produces useful knowledge per unit cost, not replace existing capture.
- (idea by user) The system should generate YouTube discovery queries from knowledge gaps, with operators able to inspect them.
- (insight by user) YouTube Data API cannot retrieve transcripts for videos the operator does not own, so transcript acquisition requires a technically and policy-safe alternative.
- (idea by coach) Generate operator-visible query proposals from coverage gaps, freshness risk, unresolved conflicts, and traveler-demand signals; model discovery relevance independently from any transcript acquisition strategy.
- (decision by user) Discovery should enrich YouTube candidates with channel subscriber count and video comments, send bounded data to AI for evaluation, and automatically escalate candidates over a threshold to Gemini video analysis; operator intervenes only for exceptional cases.
- (insight by coach) Comments are untrusted community signals, not source evidence: sanitize and bound them before triage, use them for demand/contradiction/first-hand indicators, and never let prompt-injected comment text determine publication or enter traveler retrieval.
- (decision by user) YouTube comments are used only as scoring signals, never as evidence. Discovery should run periodically from gaps/freshness priorities, with an operator kill switch.
- (idea by coach) Separate scheduled discovery from capture: the scheduler generates/enriches/triages candidates under quotas; an independent policy auto-escalates only eligible high-score candidates, while an operator kill switch stops new discovery and captures without changing completed knowledge.
- (decision by user) High-quality candidates blocked by quota or transient Gemini/API failures should remain prioritized for the next scheduled run, not alert the operator as individual exceptions.
- (insight by user) Because AI-first processes run automatically, operators need a single place to see what the overall system is doing.
- (technique) started Solution Matrix
- (idea by coach) Create an operations control tower rather than a notification feed: it shows pipeline health, throughput, queues, spending, coverage progress, and only actionable exceptions, with drill-down to run/candidate/source/card evidence.
- (decision by user) The operations control tower must serve two equally important operating needs: knowledge coverage/outcomes and automation health/cost.
- (idea by coach) Provide one control tower with two first-class views, Knowledge Mission and Automation Health, sharing an action-required queue and run-level drill-down rather than forcing either need into a secondary tab.
- (insight) Chosen direction: an AI-first periodic YouTube discovery system reuses the canonical Knowledge pipeline, treats comments only as triage signals, auto-captures eligible high-priority videos within budget, defers routine quota/failure work by priority, and exposes coverage plus automation health through an operator control tower. The design must align YouTube/Facebook policy documents with canonical AI-first publication before epic creation.
