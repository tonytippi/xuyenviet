### What Went Well

1. **Fail-closed discipline held across every protected seam.** Drizzle fails closed on missing `DATABASE_URL` (1.1). `getAuthenticatedSession()` returns `null` instead of 500 when session storage throws (1.3). AI Ask gate blocks before any side effect (1.2). Admin guard in `layout.tsx` inherits to all future `/admin/*` (1.4). Every protected path defaults to "no."
2. **Prophetic `ref` preservation in 1.2.** Story 1.2's high-severity review patch preserved `ref` across `/` → `/sign-in` → `/ai-ask` before Story 1.7 was planned. When 1.7 (referral attribution) landed, no retrofit was needed.
3. **Schema constraints over app-only checks.** `user_roles_role_check` at migration level (1.4). Audited mutations + audit inserts wrapped in shared Drizzle transaction (1.5) — cannot commit protected change without its audit row.
4. **Review caught a real ordering bug in 1.7.** Referral capture moved from Auth.js `signIn` callback (pre-persistence) to `events.signIn` hook (post-persistence) so `isNewUser` has a user ID. Security hardening followed: referral code stripped from post-auth redirect URL, HTTP-only cookie, cookie not cleared on transient exceptions.
5. **Vietnamese-first + accessibility held.** Diacritics fixed in 1.1 review, amber contrast darkened to WCAG AA, 44px touch targets, route green primary, no reward/booking/payment/maps UI leaked.
6. **Mid-epic referral insertion handled cleanly.** Referral attribution became Story 1.7 and shipped without rework.
7. **No scope creep into PRD non-goals.** Zero booking, rewards, credits, maps, affiliate, or payout behavior introduced.
8. **`deferred-work.md` is honest.** 1.6 env-guard tests explicitly blocked on test framework, not hand-waved.
9. **Traceability intact.** PRD → architecture → epics → story specs chain holds across all 7 stories.

### Challenges

1. **No test framework — deferred 5 times across 1.2, 1.3, 1.4, 1.5, 1.6.** Architecturally defensible for one story; a systemic pattern across five. Auth gate, role checks, audit transaction, env guards, referral attribution all rely on lint/typecheck/build + review eyes only. Unowned in `deferred-work.md`.
2. **Review-as-safety-net for ordering bugs.** 1.7's persistence-ordering bug was caught in review, not implementation. Epic 2's AI usage recording (2.3) has the same shape: record a usage event depending on a persisted conversation/message ID.
3. **Zero runtime/end-to-end verification until retrospective.** All 7 stories marked done on static checks. No human had signed in with Google. Partially resolved at retro: Tony confirmed OAuth callback → session → authenticated state works against real Google on localhost.
4. **Contract drift mid-epic.** `de61e1c Fix: AI Gateway URL` touched architecture, PRD, UX, and epics docs simultaneously. Contract was wrong during 1.2 and 1.3 implementation. Story 1.6 then added `assertProductionLaunchEnv()` enforcing AI Gateway keys at protected runtime on top of a URL corrected hours earlier. Near-miss: if 1.6 had shipped before the fix, production would have enforced a wrong URL.
5. **Audit coverage opt-in by design (1.5 deferred).** Future protected mutations must explicitly opt into the audited wrapper or they will not be audited.
6. **Privacy lifecycle gap (1.5, 1.7 deferred).** User deletion/anonymization behavior for audit rows and referral attribution undefined.
7. **Single-day execution.** 7 stories in one compressed day raises a depth question, not just a velocity question. Runtime verification of the core auth path at retro partially de-risked this.

### Key Insights

- **Fail-closed defaults + schema-level constraints > app-only checks.** Repeat this pattern in Epic 2 for conversation ownership and AI usage recording.
- **Review is working but should not be the only ordering-bug catcher.** Implementation needs to model callback/persistence lifecycle explicitly, especially for 2.3 (AI usage event after conversation/message persistence).
- **Runtime smoke testing is non-negotiable before building on a foundation.** Static checks passed for 7 stories; the first real OAuth run still happened only at retro.
- **Contract drift is silent.** The AI Gateway URL fix crossed 4 planning docs. Need a single source of truth or a contract-check step when fixing foundational URLs/keys.
- **Deferral without ownership is debt accrual.** `deferred-work.md` 1.6 entry had no owner, no trigger, no deadline for 5 stories. Story 2.0 now absorbs it.
